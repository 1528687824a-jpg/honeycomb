# Platform Boundaries

This project builds a downloadable multi-agent platform around OpenClaw. The
boundary must stay explicit so future packaging and open-source work do not mix
platform code with third-party product code.

## Owned By This Platform

```text
apps/
packages/
scripts/
platform-assets/openclaw-agent-templates/
docs/
```

These files are safe to develop, version, publish, and refactor as part of this
platform.

## External Runtime

OpenClaw and ClawPanel are external products. This platform should call OpenClaw
through:

```text
apps/dbos-worker/src/adapters/openclaw.ts
OPENCLAW_AGENT_MODE=real
OPENCLAW_CLI
OPENCLAW_WSL_DISTRO
```

The platform should not depend on editing OpenClaw or ClawPanel source files.

## Agent Templates

`platform-assets/openclaw-agent-templates/` contains prompt and example config
assets. They are not OpenClaw source code. They may be copied into OpenClaw
agents during setup or generated by the future M3 configuration pipeline.

## Vendor Workarounds

`platform-assets/vendor-workarounds/openclaw/` is for manual compatibility
workarounds only. Files there must not be part of the default install path for
downloadable users.

If a workaround is still needed for production, prefer one of these paths:

```text
1. Implement the behavior in this platform's adapter/config layer.
2. Use an official OpenClaw extension/configuration point.
3. Upstream a fix to OpenClaw.
4. Keep a clearly marked manual workaround for local development only.
```

## Model Switching

One-click model switching belongs in the platform's configuration layer:

```text
AgentClusterConfig
ModelProfile
OpenClaw runtime adapter
```

It should not require patching OpenClaw or ClawPanel.
